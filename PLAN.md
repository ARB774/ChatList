# PLAN.md

## Поэтапный порядок реализации программы ChatList

### Этап 1. Подготовка проекта
- Создать базовую структуру проекта: `main.py`, `db.py`, `models.py`, `network.py`.
- Подключить зависимости: `PyQt`, `SQLite`, библиотеку HTTP-запросов (`requests` или `httpx`), загрузку переменных окружения из `.env`.
- Определить единый формат конфигурации и список переменных окружения для API-ключей.

### Этап 2. Проектирование базы данных
- Создать модуль `db.py`, который инкапсулирует всю работу с `SQLite`.
- Реализовать инициализацию базы данных при первом запуске.
- Создать таблицы `prompts`, `models`, `results`, `settings`.
- Добавить индексы для часто используемых полей, например даты, имени модели и признака активности.

### Этап 3. Реализация слоя данных
- В `db.py` реализовать функции для создания схемы БД, чтения, добавления, обновления и удаления записей.
- Добавить методы получения списка сохранённых промтов.
- Добавить методы получения списка активных моделей.
- Добавить методы сохранения выбранных результатов в таблицу `results`.
- Добавить методы чтения и изменения настроек программы.

### Этап 4. Реализация работы с моделями
- В `models.py` описать структуру модели нейросети: имя, API URL, идентификатор модели, имя переменной окружения с ключом, признак активности.
- Реализовать загрузку активных моделей из БД.
- Подготовить единый интерфейс вызова модели, чтобы можно было подключать OpenAI, DeepSeek, Groq и другие сервисы.

### Этап 5. Реализация сетевого слоя
- В `network.py` реализовать отправку промта в API выбранной модели.
- Добавить формирование заголовков, получение API-ключа из `.env` по имени переменной из таблицы `models`.
- Реализовать обработку ошибок сети, таймаутов и некорректных ответов API.
- Возвращать результат в едином формате: модель, текст ответа, статус запроса, текст ошибки при наличии.

### Этап 6. Реализация временной таблицы результатов
- Создать временную структуру результатов в памяти, а не в `SQLite`.
- После ввода промта очищать старую временную таблицу.
- После получения ответов от моделей заново заполнять временную таблицу строками вида: название модели, текст ответа, флаг `selected`.
- Обеспечить возможность выбора строк пользователем через чекбоксы.

### Этап 7. Реализация графического интерфейса
- В `main.py` создать главное окно приложения на `PyQt`.
- Добавить поле ввода нового промта.
- Добавить список или таблицу сохранённых промтов с возможностью выбора.
- Добавить кнопку отправки запроса.
- Добавить таблицу результатов с колонками: модель, ответ, `selected`.
- Добавить кнопку сохранения выбранных результатов.

### Этап 8. Связка интерфейса и логики
- При выборе или вводе промта запускать процесс отправки запроса во все активные модели.
- Получать список активных моделей из БД.
- Для каждой модели вызывать сетевой слой и собирать ответы во временную таблицу.
- Отображать результаты в интерфейсе после завершения запросов.
- При нажатии на кнопку сохранения переносить только отмеченные строки в таблицу `results`.

### Этап 9. Настройка начальных данных
- Добавить стартовое заполнение таблицы `models` примерами моделей.
- Добавить экран или форму для управления моделями: включение, отключение, редактирование URL и идентификатора.
- Добавить экран или форму для изменения базовых настроек программы.

### Этап 10. Поиск, сортировка и дополнительные функции
- Реализовать поиск и сортировку во всех таблицах интерфейса.
- Добавить экспорт выбранных результатов в `Markdown` и `JSON`.
- При необходимости добавить логирование запросов и ответов.

### Этап 11. Тестирование
- Проверить создание базы данных с нуля.
- Проверить выбор сохранённого промта и ввод нового промта.
- Проверить отправку одного запроса в несколько активных моделей.
- Проверить отображение временной таблицы и очистку при новом запросе.
- Проверить сохранение только отмеченных результатов в `results`.
- Проверить обработку ошибок API и отсутствие падения интерфейса.

### Этап 12. Сборка и выпуск
- Подготовить приложение к запуску на целевой системе.
- Настроить сборку исполняемого файла.
- Проверить работу программы после сборки с использованием реальной базы данных и `.env`.
